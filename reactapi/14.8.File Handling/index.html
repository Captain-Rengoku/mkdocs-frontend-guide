    File Handling

    Handling files is one of the most common — and most error-prone — parts of building a real app: profile pictures, document uploads, CSV exports, PDF previews, video streams. This chapter covers the browser primitives (File, Blob, FormData), then wires them up to axios v1 for uploads and downloads with progress, validation, cancellation, chunked/resumable transfers, and direct-to-cloud flows — finishing with a reusable useFileUpload hook.

    Builds on the Axios chapter

    The Axios chapter already introduced basic uploadFile / downloadFile with onUploadProgress and responseType: "blob". Here we go deeper: typed progress events, validation, previews, chunking, streaming, abort, and presigned URLs. The patterns stay consistent — same AxiosProgressEvent, same divide-by-zero guard, same object-URL revocation.

    1️⃣ Browser File Primitives

    Before talking to any API, you need to understand the objects the browser hands you.

    File, Blob & FileList

    • Blob — immutable raw binary data (size, type). The base building block.
    • File — a Blob with extra metadata (name, lastModified). This is what <input type="file"> gives you.
    • FileList — array-like collection of File objects (from input.files or a drop event). Not a real array — convert with Array.from().
    const input = document.querySelector<HTMLInputElement>("#file")!;
const files: FileList | null = input.files;

if (files) {
  const list: File[] = Array.from(files);
  list.forEach((file) => {
    console.log(file.name);          // "photo.png"
    console.log(file.size);          // bytes
    console.log(file.type);          // "image/png"
    console.log(file.lastModified);  // timestamp
  });
}

    Creating Blobs & Files manually

    Useful for generating downloads, slicing for chunked uploads, or building test fixtures.

    // A text Blob
const blob = new Blob(["id,name\n1,Ada\n2,Linus"], { type: "text/csv" });

// A File from a Blob (e.g., a generated report)
const file = new File([blob], "users.csv", { type: "text/csv" });

// Slicing — the core of chunked uploads
const firstChunk = file.slice(0, 1024 * 1024, file.type); // first 1 MB

    Reading file contents

    Modern code prefers the promise-based Blob / File methods over the old FileReader:

    declare const file: File;

const text = await file.text();             // string
const buffer = await file.arrayBuffer();    // ArrayBuffer
const stream = file.stream();               // ReadableStream

    FileReader is legacy

    FileReader (with onload / readAsDataURL) still works and is needed for some older targets, but file.text() / file.arrayBuffer() are cleaner. For previews, prefer URL.createObjectURL() (below) over base64 data URLs — it's faster and uses far less memory.

    2️⃣ File Input & Drag-and-Drop

    Controlled file input

    A file input is uncontrolled for its value (you cannot set value programmatically for security reasons), so track the selected files in state.

    import { useState, ChangeEvent } from "react";

function FilePicker() {
  const [files, setFiles] = useState<File[]>([]);

  const handleChange = (e: ChangeEvent<HTMLInputElement>) => {
    if (e.target.files) {
      setFiles(Array.from(e.target.files));
    }
  };

  return (
    <div>
      <input
        type="file"
        multiple
        accept="image/*,.pdf"
        onChange={handleChange}
      />
      <ul>
        {files.map((f) => (
          <li key={f.name}>
            {f.name} — {(f.size / 1024).toFixed(1)} KB
          </li>
        ))}
      </ul>
    </div>
  );
}
    • multiple allows selecting more than one file.
    • accept is a hint to the OS file dialog (image/*, .pdf, application/pdf). Never trust it for validation — always re-check in JS (and on the server).

    Drag-and-drop drop zone

    A drop zone needs three handlers: onDragOver (must preventDefault to allow a drop), onDragLeave, and onDrop.

    import { useState, DragEvent } from "react";

function DropZone({ onFiles }: { onFiles: (files: File[]) => void }) {
  const [isOver, setIsOver] = useState(false);

  const handleDragOver = (e: DragEvent<HTMLDivElement>) => {
    e.preventDefault();        // REQUIRED — otherwise the browser opens the file
    setIsOver(true);
  };

  const handleDragLeave = (e: DragEvent<HTMLDivElement>) => {
    e.preventDefault();
    setIsOver(false);
  };

  const handleDrop = (e: DragEvent<HTMLDivElement>) => {
    e.preventDefault();
    setIsOver(false);
    if (e.dataTransfer.files?.length) {
      onFiles(Array.from(e.dataTransfer.files));
    }
  };

  return (
    <div
      onDragOver={handleDragOver}
      onDragLeave={handleDragLeave}
      onDrop={handleDrop}
      style={{
        border: `2px dashed ${isOver ? "#2563eb" : "#cbd5e1"}`,
        borderRadius: 8,
        padding: 32,
        textAlign: "center",
        background: isOver ? "#eff6ff" : "transparent",
      }}
    >
      Drag &amp; drop files here, or click to browse
    </div>
  );
}

    Always preventDefault on dragover

    If you forget e.preventDefault() inside onDragOver, the onDrop event will never fire — the browser will instead navigate to / open the dropped file. This is the single most common drag-and-drop bug.

    3️⃣ FormData — the upload envelope

    multipart/form-data is how browsers send files. FormData builds that payload for you.

    const formData = new FormData();

// Append a single file (3rd arg overrides the filename sent to the server)
formData.append("avatar", file, file.name);

// Append regular fields alongside files
formData.append("userId", "42");
formData.append("title", "My upload");

// Append MANY files under the same key
files.forEach((f) => formData.append("documents", f));
// → server receives an array under "documents"

    Let axios set the boundary

    With axios v1 and a browser FormData, do not manually set Content-Type: multipart/form-data. Axios detects FormData and sets the header including the required boundary for you. Setting it by hand drops the boundary and the server can't parse the body. (The Axios chapter set it explicitly for illustration; in practice, omit it and let axios handle it.)

    4️⃣ Uploading Files with axios v1

    A shared API client

    // api.ts
import axios from "axios";

export const api = axios.create({
  baseURL: import.meta.env.VITE_API_URL ?? "https://api.example.com",
  withCredentials: true,
});

    Single file upload

    import { api } from "./api";

interface UploadResult {
  id: string;
  url: string;
}

async function uploadAvatar(file: File): Promise<UploadResult> {
  const formData = new FormData();
  formData.append("avatar", file);

  const { data } = await api.post<UploadResult>("/upload/avatar", formData);
  // No manual Content-Type — axios sets multipart + boundary automatically.
  return data;
}

    Multiple file upload

    async function uploadDocuments(files: File[]): Promise<UploadResult[]> {
  const formData = new FormData();
  files.forEach((file) => formData.append("documents", file));

  const { data } = await api.post<UploadResult[]>("/upload/documents", formData);
  return data;
}

    One request vs. many

    Appending every file to a single FormData sends one request — simple, but a failure loses the whole batch and you only get one combined progress number. Uploading each file in its own request (e.g. Promise.all(files.map(uploadOne))) gives per-file progress and isolated retries. Choose based on UX needs.

    5️⃣ Upload Progress with onUploadProgress

    Axios reports progress through onUploadProgress, typed as AxiosProgressEvent. Always guard the percentage math against a missing/zero total (division by zero → NaN / Infinity).

    import { api } from "./api";
import type { AxiosProgressEvent } from "axios";

async function uploadWithProgress(
  file: File,
  onProgress: (percent: number) => void
) {
  const formData = new FormData();
  formData.append("file", file);

  const { data } = await api.post("/upload", formData, {
    onUploadProgress: (event: AxiosProgressEvent) => {
      // Guard: total can be undefined (unknown size) or 0
      const total = event.total ?? 0;
      if (total <= 0) return;
      const percent = Math.round((event.loaded * 100) / total);
      onProgress(percent);
    },
  });

  return data;
}

    AxiosProgressEvent also exposes event.progress (0–1, already computed), event.rate (bytes/sec), and event.estimated (seconds remaining) when available — handy for speed/ETA UIs:

    onUploadProgress: (event: AxiosProgressEvent) => {
  const percent = Math.round((event.progress ?? 0) * 100); // 0–1 → %
  const kbps = event.rate ? Math.round(event.rate / 1024) : null;
  const eta = event.estimated ? Math.ceil(event.estimated) : null;
  console.log(`${percent}%  ${kbps ?? "?"} KB/s  ETA ${eta ?? "?"}s`);
}

    A progress bar component

    import { useState } from "react";

function UploadField() {
  const [percent, setPercent] = useState(0);
  const [busy, setBusy] = useState(false);

  const handleUpload = async (file: File) => {
    setBusy(true);
    setPercent(0);
    try {
      await uploadWithProgress(file, setPercent);
    } finally {
      setBusy(false);
    }
  };

  return (
    <div>
      <input
        type="file"
        disabled={busy}
        onChange={(e) => e.target.files?.[0] && handleUpload(e.target.files[0])}
      />
      {busy && (
        <progress value={percent} max={100}>
          {percent}%
        </progress>
      )}
    </div>
  );
}

    6️⃣ Client-Side Validation (size & type)

    Validate before uploading to fail fast and save bandwidth. Treat this as UX only — the server must validate too (clients can be bypassed).

    const MAX_SIZE = 5 * 1024 * 1024; // 5 MB
const ALLOWED_TYPES = ["image/png", "image/jpeg", "image/webp"];

interface ValidationError {
  file: string;
  reason: string;
}

function validateFiles(files: File[]): {
  valid: File[];
  errors: ValidationError[];
} {
  const valid: File[] = [];
  const errors: ValidationError[] = [];

  for (const file of files) {
    if (file.size > MAX_SIZE) {
      errors.push({
        file: file.name,
        reason: `Too large (${(file.size / 1024 / 1024).toFixed(1)} MB, max 5 MB)`,
      });
    } else if (!ALLOWED_TYPES.includes(file.type)) {
      errors.push({ file: file.name, reason: `Unsupported type: ${file.type || "unknown"}` });
    } else {
      valid.push(file);
    }
  }

  return { valid, errors };
}

    file.type is spoofable

    The MIME type comes from the OS / extension and can lie. For sensitive flows, sniff the real type from the file's magic bytes (read the first few bytes of arrayBuffer() and compare signatures), and always re-verify on the server.

    7️⃣ Image Preview with createObjectURL

    URL.createObjectURL(file) returns a short-lived blob: URL pointing at the file in memory — ideal for instant previews without uploading or base64-encoding. Each URL holds the blob in memory until you revoke it, so revocation is mandatory to avoid leaks.

    import { useState, useEffect } from "react";

function ImagePreview({ file }: { file: File }) {
  const [url, setUrl] = useState<string>("");

  useEffect(() => {
    const objectUrl = URL.createObjectURL(file);
    setUrl(objectUrl);

    // Cleanup: revoke when file changes or component unmounts
    return () => URL.revokeObjectURL(objectUrl);
  }, [file]);

  return url ? <img src={url} alt={file.name} width={200} /> : null;
}

    For a list of previews, revoke every URL on cleanup:

    import { useState, useEffect } from "react";

function Gallery({ files }: { files: File[] }) {
  const [urls, setUrls] = useState<string[]>([]);

  useEffect(() => {
    const next = files.map((f) => URL.createObjectURL(f));
    setUrls(next);
    return () => next.forEach((u) => URL.revokeObjectURL(u));
  }, [files]);

  return (
    <div style={{ display: "flex", gap: 8, flexWrap: "wrap" }}>
      {urls.map((u, i) => (
        <img key={files[i].name} src={u} alt={files[i].name} width={120} />
      ))}
    </div>
  );
}

    Always revoke object URLs

    Forgetting URL.revokeObjectURL() leaks memory — every preview pins its file in RAM for the page's lifetime. Revoke in the useEffect cleanup (or right after the <img> loads if it's a one-shot).

    8️⃣ Canceling Uploads & Downloads (AbortController)

    axios v1 supports the standard AbortController. Pass its signal in the request config, then call controller.abort() to cancel. (The deprecated CancelToken still works but prefer AbortController — see the Axios chapter.)

    import axios, { AxiosProgressEvent } from "axios";
import { api } from "./api";

async function uploadCancelable(
  file: File,
  signal: AbortSignal,
  onProgress: (p: number) => void
) {
  const formData = new FormData();
  formData.append("file", file);

  try {
    const { data } = await api.post("/upload", formData, {
      signal,
      onUploadProgress: (e: AxiosProgressEvent) => {
        const total = e.total ?? 0;
        if (total > 0) onProgress(Math.round((e.loaded * 100) / total));
      },
    });
    return data;
  } catch (err) {
    if (axios.isCancel(err)) {
      console.warn("Upload canceled by user");
      return null;
    }
    throw err;
  }
}

    A component with a Cancel button:

    import { useRef, useState } from "react";

function CancelableUpload() {
  const [percent, setPercent] = useState(0);
  const [busy, setBusy] = useState(false);
  const controllerRef = useRef<AbortController | null>(null);

  const start = async (file: File) => {
    const controller = new AbortController();
    controllerRef.current = controller;
    setBusy(true);
    try {
      await uploadCancelable(file, controller.signal, setPercent);
    } finally {
      setBusy(false);
      controllerRef.current = null;
    }
  };

  return (
    <div>
      <input
        type="file"
        disabled={busy}
        onChange={(e) => e.target.files?.[0] && start(e.target.files[0])}
      />
      {busy && (
        <>
          <progress value={percent} max={100} />
          <button onClick={() => controllerRef.current?.abort()}>Cancel</button>
        </>
      )}
    </div>
  );
}

    Detecting cancellation

    Use axios.isCancel(err) to distinguish a deliberate abort from a real failure. The thrown error's name is "CanceledError". Always abort in-flight requests on unmount to avoid setting state on a dead component.

    9️⃣ Downloading Files (blob responses)

    To download binary data, set responseType: "blob", then turn the blob into an object URL and click a temporary anchor.

    import { api } from "./api";

async function downloadFile(url: string, filename: string) {
  const response = await api.get(url, { responseType: "blob" });

  // Prefer the server's Content-Disposition filename when present
  const disposition = response.headers["content-disposition"] as string | undefined;
  const serverName = disposition?.match(/filename="?([^"]+)"?/)?.[1];

  const blobUrl = URL.createObjectURL(response.data);
  const link = document.createElement("a");
  link.href = blobUrl;
  link.download = serverName ?? filename;
  document.body.appendChild(link);
  link.click();
  link.remove();
  URL.revokeObjectURL(blobUrl); // revoke immediately — the download is already queued
}

// downloadFile("/reports/2026-q2.pdf", "report.pdf");

    Download progress

    onDownloadProgress mirrors onUploadProgress — same AxiosProgressEvent, same guard. Note that event.total may be missing if the server omits Content-Length (e.g. chunked transfer encoding).

    import type { AxiosProgressEvent } from "axios";

async function downloadWithProgress(
  url: string,
  filename: string,
  onProgress: (p: number) => void
) {
  const response = await api.get(url, {
    responseType: "blob",
    onDownloadProgress: (event: AxiosProgressEvent) => {
      const total = event.total ?? 0;
      if (total <= 0) return; // unknown length — show a spinner instead
      onProgress(Math.round((event.loaded * 100) / total));
    },
  });

  const blobUrl = URL.createObjectURL(response.data);
  const link = document.createElement("a");
  link.href = blobUrl;
  link.download = filename;
  link.click();
  URL.revokeObjectURL(blobUrl);
}

    Generating files client-side

    The same anchor-click trick downloads files you build in the browser — no server round trip:

    function downloadCsv(rows: string[][], filename = "export.csv") {
  const csv = rows.map((r) => r.join(",")).join("\n");
  const blob = new Blob([csv], { type: "text/csv;charset=utf-8;" });
  const url = URL.createObjectURL(blob);
  const a = document.createElement("a");
  a.href = url;
  a.download = filename;
  a.click();
  URL.revokeObjectURL(url);
}

    🔟 Previewing PDFs & Images

    You can preview a downloaded blob inline rather than forcing a download.

    import { useEffect, useState } from "react";
import { api } from "./api";

function PdfViewer({ url }: { url: string }) {
  const [blobUrl, setBlobUrl] = useState<string>("");

  useEffect(() => {
    let active = true;
    let created = "";

    api
      .get(url, { responseType: "blob" })
      .then((res) => {
        // Ensure the browser treats it as a PDF
        const pdfBlob = new Blob([res.data], { type: "application/pdf" });
        created = URL.createObjectURL(pdfBlob);
        if (active) setBlobUrl(created);
      });

    return () => {
      active = false;
      if (created) URL.revokeObjectURL(created);
    };
  }, [url]);

  if (!blobUrl) return <p>Loading PDF…</p>;

  return (
    <iframe
      src={blobUrl}
      title="PDF preview"
      width="100%"
      height={600}
      style={{ border: "none" }}
    />
  );
}
    • PDFs: render in an <iframe> or <embed> pointing at the object URL. For richer control (page navigation, text layer), use a library such as pdf.js / react-pdf.
    • Images: just point an <img src> at the object URL (see §7). Same revocation rules apply.

    1️⃣1️⃣ Chunked & Resumable Uploads

    For very large files (or flaky networks), split the file into chunks, upload them one by one, and let the server reassemble. If the connection drops, you resume from the last acknowledged chunk instead of restarting.

    The concept:

    1. Slice the file into fixed-size chunks with file.slice().
    2. Upload each chunk with its index + total count + a stable upload ID.
    3. Track which chunks succeeded; on resume, skip them.
    4. After the last chunk, call a "complete" endpoint so the server stitches them together.
    import { api } from "./api";
import type { AxiosProgressEvent } from "axios";

const CHUNK_SIZE = 5 * 1024 * 1024; // 5 MB

interface ChunkUploadOptions {
  uploadId: string;            // stable id for this file (uuid)
  signal?: AbortSignal;
  onProgress?: (percent: number) => void;
  startChunk?: number;         // resume point
}

async function uploadInChunks(file: File, opts: ChunkUploadOptions) {
  const totalChunks = Math.ceil(file.size / CHUNK_SIZE);
  const start = opts.startChunk ?? 0;

  for (let index = start; index < totalChunks; index++) {
    const from = index * CHUNK_SIZE;
    const blob = file.slice(from, from + CHUNK_SIZE);

    const formData = new FormData();
    formData.append("uploadId", opts.uploadId);
    formData.append("index", String(index));
    formData.append("totalChunks", String(totalChunks));
    formData.append("chunk", blob, file.name);

    await api.post("/upload/chunk", formData, {
      signal: opts.signal,
      onUploadProgress: (e: AxiosProgressEvent) => {
        const chunkTotal = e.total ?? 0;
        if (chunkTotal <= 0) return;
        const chunkFraction = e.loaded / chunkTotal;          // 0–1 within this chunk
        const overall = (index + chunkFraction) / totalChunks; // 0–1 across the file
        opts.onProgress?.(Math.round(overall * 100));
      },
    });
  }

  // Tell the server to assemble the chunks into the final file
  const { data } = await api.post("/upload/complete", {
    uploadId: opts.uploadId,
    filename: file.name,
    totalChunks,
  });
  return data;
}

    Resuming after a failure — ask the server which chunks it already has, then restart from there:

    async function resumeUpload(file: File, uploadId: string) {
  const { data } = await api.get<{ uploadedChunks: number[] }>(
    `/upload/status/${uploadId}`
  );
  const next = data.uploadedChunks.length; // assumes sequential upload
  return uploadInChunks(file, { uploadId, startChunk: next });
}

    Production-grade resumable uploads

    Rolling your own is great for learning. For production, consider the tus protocol (tus-js-client) or Uppy, which handle chunking, resume, retries, and parallelism for you. Cloud providers (S3 multipart, GCS resumable) also expose native resumable APIs.

    1️⃣2️⃣ Streaming Large Files

    When a file is too big to hold in memory as one blob, stream it. The Fetch API exposes the response body as a ReadableStream, letting you process bytes as they arrive (and compute progress even when axios can't, since axios buffers the whole response).

    async function streamDownload(url: string, onProgress: (p: number) => void) {
  const response = await fetch(url);
  if (!response.body) throw new Error("ReadableStream not supported");

  const total = Number(response.headers.get("Content-Length")) || 0;
  const reader = response.body.getReader();
  const chunks: Uint8Array[] = [];
  let received = 0;

  while (true) {
    const { done, value } = await reader.read();
    if (done) break;
    if (value) {
      chunks.push(value);
      received += value.length;
      if (total > 0) onProgress(Math.round((received * 100) / total)); // guard
    }
  }

  // Reassemble into a Blob once fully read
  return new Blob(chunks);
}

    For uploads, the modern (Chromium) approach streams a ReadableStream request body via fetch with duplex: "half". Browser support is still uneven, so chunked uploads (§11) remain the portable choice.

    Why stream?

    Streaming keeps memory flat regardless of file size, lets you start processing/rendering before the full file arrives, and powers media playback. axios buffers the whole response into a blob, so reach for fetch + ReadableStream when you specifically need incremental access.

    1️⃣3️⃣ Uploading to Cloud Storage (Presigned URLs)

    For large files, and to keep traffic off your own servers, upload directly to object storage (S3, GCS, R2, Azure Blob). Your backend stays in control by issuing a short-lived presigned URL; the browser then PUTs the file straight to the bucket.

    The S3-style flow:

    1. Client asks your API for a presigned URL (sends filename + content type).
    2. Your server signs a time-limited PUT URL and returns it.
    3. Client uploads the file directly to that URL (with progress).
    4. Client tells your API the upload finished so it can record the object key.
    import axios, { AxiosProgressEvent } from "axios";
import { api } from "./api";

interface PresignResponse {
  uploadUrl: string; // signed PUT URL on the bucket
  key: string;       // object key to store in your DB
}

async function uploadToS3(file: File, onProgress: (p: number) => void) {
  // 1. Get a presigned URL from YOUR backend
  const { data: presign } = await api.post<PresignResponse>("/uploads/presign", {
    filename: file.name,
    contentType: file.type,
  });

  // 2. PUT the raw file straight to the bucket.
  //    Use a bare axios call (not `api`) so your auth headers / baseURL aren't sent
  //    to the storage provider. Send the raw File, NOT FormData.
  await axios.put(presign.uploadUrl, file, {
    headers: { "Content-Type": file.type }, // must match what was signed
    onUploadProgress: (e: AxiosProgressEvent) => {
      const total = e.total ?? 0;
      if (total > 0) onProgress(Math.round((e.loaded * 100) / total));
    },
  });

  // 3. Confirm completion with your backend
  await api.post("/uploads/confirm", { key: presign.key });

  return presign.key;
}

    Presigned upload gotchas

    • Send the raw File with PUT, not FormData — S3 presigned PUT expects the file as the request body.
    • The Content-Type you send must exactly match the one used when signing, or the bucket rejects it (SignatureDoesNotMatch).
    • Use a plain axios instance (or strip interceptors) for the storage PUT so your app's Authorization header isn't leaked to the third-party host.
    • Configure CORS on the bucket to allow PUT from your origin, or the browser blocks the request.
    • Presigned URLs expire — request them right before uploading, not at page load.

    1️⃣4️⃣ A Reusable useFileUpload Hook

    Pulling it all together: a typed hook with validation, progress, cancellation, and clean error handling.

    // useFileUpload.ts
import { useCallback, useRef, useState } from "react";
import axios, { AxiosProgressEvent, AxiosError } from "axios";
import { api } from "./api";

type UploadStatus =
  | "idle"
  | "validating"
  | "uploading"
  | "success"
  | "error"
  | "canceled";

interface UseFileUploadOptions {
  url: string;
  fieldName?: string;          // form field key (default "file")
  maxSize?: number;            // bytes
  allowedTypes?: string[];     // MIME types
}

interface UseFileUploadReturn<T> {
  status: UploadStatus;
  progress: number;            // 0–100
  error: string | null;
  data: T | null;
  upload: (file: File) => Promise<T | null>;
  cancel: () => void;
  reset: () => void;
}

export function useFileUpload<T = unknown>(
  options: UseFileUploadOptions
): UseFileUploadReturn<T> {
  const { url, fieldName = "file", maxSize, allowedTypes } = options;

  const [status, setStatus] = useState<UploadStatus>("idle");
  const [progress, setProgress] = useState(0);
  const [error, setError] = useState<string | null>(null);
  const [data, setData] = useState<T | null>(null);
  const controllerRef = useRef<AbortController | null>(null);

  const validate = useCallback(
    (file: File): string | null => {
      if (maxSize && file.size > maxSize) {
        return `File too large (max ${(maxSize / 1024 / 1024).toFixed(1)} MB)`;
      }
      if (allowedTypes && !allowedTypes.includes(file.type)) {
        return `Unsupported file type: ${file.type || "unknown"}`;
      }
      return null;
    },
    [maxSize, allowedTypes]
  );

  const upload = useCallback(
    async (file: File): Promise<T | null> => {
      setStatus("validating");
      setError(null);
      setData(null);
      setProgress(0);

      const validationError = validate(file);
      if (validationError) {
        setError(validationError);
        setStatus("error");
        return null;
      }

      const controller = new AbortController();
      controllerRef.current = controller;
      setStatus("uploading");

      const formData = new FormData();
      formData.append(fieldName, file);

      try {
        const response = await api.post<T>(url, formData, {
          signal: controller.signal,
          onUploadProgress: (e: AxiosProgressEvent) => {
            const total = e.total ?? 0;
            if (total <= 0) return;            // divide-by-zero guard
            setProgress(Math.round((e.loaded * 100) / total));
          },
        });
        setData(response.data);
        setStatus("success");
        return response.data;
      } catch (err) {
        if (axios.isCancel(err)) {
          setStatus("canceled");
          return null;
        }
        const axiosErr = err as AxiosError<{ message?: string }>;
        setError(
          axiosErr.response?.data?.message ?? axiosErr.message ?? "Upload failed"
        );
        setStatus("error");
        return null;
      } finally {
        controllerRef.current = null;
      }
    },
    [url, fieldName, validate]
  );

  const cancel = useCallback(() => {
    controllerRef.current?.abort();
  }, []);

  const reset = useCallback(() => {
    controllerRef.current?.abort();
    controllerRef.current = null;
    setStatus("idle");
    setProgress(0);
    setError(null);
    setData(null);
  }, []);

  return { status, progress, error, data, upload, cancel, reset };
}

    Using the hook

    import { useFileUpload } from "./useFileUpload";

interface UploadResult {
  id: string;
  url: string;
}

function AvatarUploader() {
  const { status, progress, error, data, upload, cancel, reset } =
    useFileUpload<UploadResult>({
      url: "/upload/avatar",
      fieldName: "avatar",
      maxSize: 5 * 1024 * 1024,
      allowedTypes: ["image/png", "image/jpeg", "image/webp"],
    });

  return (
    <div>
      <input
        type="file"
        accept="image/*"
        disabled={status === "uploading"}
        onChange={(e) => e.target.files?.[0] && upload(e.target.files[0])}
      />

      {status === "uploading" && (
        <>
          <progress value={progress} max={100} /> {progress}%
          <button onClick={cancel}>Cancel</button>
        </>
      )}

      {status === "error" && <p style={{ color: "crimson" }}>{error}</p>}
      {status === "canceled" && <p>Upload canceled.</p>}
      {status === "success" && data && (
        <>
          <img src={data.url} alt="Uploaded avatar" width={120} />
          <button onClick={reset}>Upload another</button>
        </>
      )}
    </div>
  );
}

    ✅ Best Practices Recap

    • Let axios set Content-Type for FormData — never hardcode the boundary.
    • Always guard progress math against a missing/zero total (event.total ?? 0, bail if <= 0).
    • Revoke every object URL (URL.revokeObjectURL) in cleanup to prevent memory leaks.
    • Validate on the client for UX, on the server for security — file.type and accept are spoofable.
    • Abort in-flight requests on unmount with AbortController; use axios.isCancel() to tell cancellation from failure.
    • Chunk large uploads (or use tus/Uppy) so failures resume instead of restarting.
    • Stream with fetch + ReadableStream when files won't fit in memory.
    • Upload large files directly to cloud via presigned URLs — keep the bytes off your own servers, and never leak your auth headers to the storage host.
    • Prefer object URLs over base64 data URLs for previews — they're faster and far lighter on memory.
